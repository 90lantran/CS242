Assignment 3.0 - Creating a Portfolio

Overview

This is the first week of a multi-week assignment where you will be building a portfolio to show off all of the code that you have written over the duration of the the semester. We will start this will with basic PHP, HTML, and CSS, and move on to other web technologies such as JavaScript and SQL in future weeks.

For this week, you will parse information about your submissions to SVN and use it to create a simple, static portfolio site with some basic styling.

Don't fall behind!

This week is the first of several for this assignment and contains several different technologies, many of which may be unfamiliar to you. Start early, make sure you understand the task and hand and get your development environment setup as soon as possible, so that if you run into problems, you can get unstuck early.

While you are not required to setup a development server on your own computer (i.e. such as Apache), it is highly encouraged. For a walkthrough of how you may setup your development environment including PHP & Apache, see Getting Started with Web Development.

Regardless of where you develop your code, by the time of discussion section, you should also deploy your project to cpanel: https://web.engr.illinois.edu/. For more information on using cpanel & hosting projects using the University's resources, see: http://it.engineering.illinois.edu/user-guides/how-do-i-get-a-website. Your project should be deployed to:

http://web.engr.illinois.edu/~[netid]/

Part I: Tutorials

This is listed as a separate part to the assignment, because it can take some time if you have never used PHP, HTML, & CSS before. First, familiarize yourself with the basics of HTML & CSS. W3Schools as a good place to start, but you may find more in-depth tutorials useful as well:

  • W3Schools HTML Tutorial
  • W3Schools CSS Tutorial
  • Google's HTML, CSS, and Javascript from the Ground Up Tutorial

Be sure to try out what you've learned with a few examples, and once you have a basic understanding of HTML & CSS, and you can create a very basic site manually, move on to learning the basics of PHP.

PHP is a scripting language that is particularly useful for web development, because you can create website easily without any complex server configurations. We require this language because it is used widely, and makes your work this week easy to extend to following weeks.

To build a basic understanding of PHP, take a look at the W3Schools PHP tutorial. There are plenty of high quality resources for PHP online as well, including the official PHP manual, which can be particularly valuable for finding information about a particular function.

Setup Your Development Environment

Before moving on, be sure to setup your development environment and create a few basic examples on your own. If you have trouble setting up your development environment, follow the Getting Started with Web Development guide incrementally, checking that your environment is correct after each step.

I would recommend taking a look at an IDE, for instance PHPStorm (from the makers of IntelliJ IDEA, free for 30 days – longer than our assignment). Eclipse likewise has PHP support.

Part II: Parse SVN Data

For this section, you will need to get access to a machine that has the svn command line program on it. The EWS linux machines have it already installed. To get the raw data, enter the following command:

svn log --verbose --xml https://subversion.ews.illinois.edu/svn/fa12-cs242/{netid} > svn_log.xml
svn list --xml --recursive https://subversion.ews.illinois.edu/svn/fa12-cs242/{netid} > svn_list.xml

This will save the log and listing of your repository into a file called svn_log.xml and svn_list.xml. See the svn log documentation or run the command svn help log to get more details on the svn log command. See similar pages for svn list help.

Design Considerations
You may find that elegant solutions to parsing the XML from SVN may be similar to common solutions for compilers or similar applications (i.e. it may be worth considering existing related design patterns, such as Interpreter, Composite, or Visitor, etc.).

Ultimately, it's up to you on how to design your parsing and PHP data structures, but spend some time considering a good design. Keep in mind that PHP is object-oriented, and so we expected a well thought out and (likely) object-oriented design.

Part III: Create a Basic Portfolio

Next, using the parsed data from SVN, create a static website using this content. It is up to you on how to design your website, but it should contain content about your projects and use CSS for a consistent, stylish theme.

Style / Theme

You are welcome to write your own CSS by hand if you would like to make sure you learn CSS, but using an existing style template from an online source is preferred, since it is likely higher quality than anything you would write in a reasonable amount of time, and numerous templates are available online. Although a minimalist theme is perfectly acceptable, your theme must use a nontrivial amount of CSS (i.e. 5-10 lines of CSS is likely not acceptable).

There are thousands of templates that you could use to start this portion of the assignment, but if you download source code from an online source (or any source), cite it. This is particularly important for this part of the assignment!

Likewise, if you use a template, do not leave filler content in the template that you use. Your site should have your own content only. This means that you must strip out any content that initially comes with the template, and adapt the template to your application.

Again, there are numerous choices, but for someone just learning web development, Free CSS Templates is an excellent resource for free, high quality themed templates.

Likewise, for more advanced or the above average students, HTML 5 Boilerplate, Bootstrap or similar modern CSS frameworks are excellent choices or libraries to supplement your theme. They provide valuable tools to make your site responsive to different screen sizes, add common UI components, and give your site a polished feel very quickly.

This list of resources if far from comprehensive, and if you think there should be other resources added to this list, contact the course staff with your suggestion(s).

Functional Requirements

The structure of your website is up to you, but it should contain the following data:

  • The title of each of your projects
  • The date for each project
  • The version for each project
  • The summary for each project (the most recent commit message for that assignment)
  • A listing of each file in the project with
    • Its size
    • The type of the file: is one of code, test, image, documentation, resource, etc (feel free to add as many types as you wish).
    • The path is the path to your files in SVN.
    • The file itself loaded in an iframe only on demand
    • Each version of each file in the project
      • The number is the revision number for that commit
      • The author is the netid of the committer
      • The info is the commit message for that revision
      • The date is the date of that commit

Put some time into your site layout, and consider it as you choose the template and/or write the CSS to use for your site theme.

Testing

As always, testing is an important part of this week's assignment, particularly for the parsing portion of the assignment.

At minimum, you should be able to construct several small input XML files by hand and verify that your code produces the proper data structure in memory, then outputs the proper converted XML back to the user. You should test for all possible corner cases you can think of, such as:

  • multiple revisions for a file
  • multiple revisions for multiple files
  • files in conflicted SVN states
  • files that were SVN merged
  • files that were deleted
  • etc...

PHP has many options for unit testing, the most widely used of which are likely PHPUnit and SimpleTest. For more information, see this conversation. Your tests, as always, are expected to be automated and should use some unit testing framework.

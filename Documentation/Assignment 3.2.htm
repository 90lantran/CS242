Assignment 3.2 - Modernizing Your Portfolio

Overview

This week, you will be putting together your final project proposal and working to make your portfolio dynamic. Specifically, we will be focus on adding AJAX to a few key areas of your portfolio, as well as including syntax highlighting for your project source code.

Part I: Final Project Proposal

For the last four weeks of the semester, you will work on a Final Project of your choice. For this week, you must submit a project proposal electronically as a PDF to your moderator by Monday, October 29 at 3pm.

There are no restrictions on what language, frameworks, or tools that you use, but your project:

  • Cannot be any work you are getting paid for
  • Cannot be any project or assignment for another course you are currently taking
  • Must have clear, concrete objectives for each week
  • Must include tools for testing, as this will be part of each week's requirements

The proposal should be a PDF containing a summary of the project, your motivation for the project, and specific requirements for each week. Specifically, your proposal should include:

  • Description/summary of project
  • Your motivation for choosing this project
  • Challenges
  • List of overall features
  • Concrete project plan

We realize that it is difficult to plan your project week by week without actually implementing it, but this is a key part of the design process.

In terms of your grade, you cannot drop any week of the final project!.

In addition to individual final projects, we will allow

  • Group projects - only for certain projects, and with special approval from the TAs or professor Woodley
  • Open Source or existing projects - again, only subject to special approval

With either of these special cases, you must very clearly define your contributions specifically. For group projects, all moderators involved must be emailed the same proposal, and this proposal should clearly define the contributions of each member individually.

If you are interested in participating in the Google programming competition mentioned during lecture a few weeks ago, see the guidelines attached. They should be completely compatible with our project guidelines.

Updating Your Proposal
As you implement your project each week, your actual progress may differ from your planned progress. We understand that this may happen, and as you implement your project each week, you should update your proposal as well. This may include any changes to technologies for your project. Stay in communication with your moderator – ask before making in drastic changes to your proposal in following weeks, and send him or her any updates to your proposal.

Part II: Add jQuery to Your Portfolio

To ensure that the following parts of the assignment are not more difficult than they need to be, we will expect you to use jQuery specifically for this week's assignment. This is an extremely robust & widely used library, and is essentially a JavaScript utility library for manipulating the DOM and provided common functions (including AJAX).

The homepage of jQuery can be found here:

  • http://jquery.com/

Further examples and documentation on jQuery are available from the homepage, wikipedia, and w3 schools:

  • http://docs.jquery.com/How_jQuery_Works
  • http://en.wikipedia.org/wiki/JQuery
  • http://www.w3schools.com/jquery/jquery_examples.asp

Including jQuery to your portfolio should be trivial. Since it is just a JavaScript file, adding a single include line to the <head> ... </head> tag of your site will do the trick. If you download the jQuery source file as jquery.js, then the following line in the <head> ... </head> tag of your site will suffice:

Installing jQuery to Your Site
<script type="text/javascript" src="jquery.js"></script>

You must use some feature of jQuery on your site to get credit for this part. This can be as simple as performing a basic animation or manipulation of the DOM, use it in some way on your site before moving on, so that you understand how it works.

Part III: Post Comments Asynchronously

Using jQuery, and this week's lecture, you should be able to implement a basic AJAX request. For this portion of the assignment, use AJAX to allow a user to post a comment & see it appear on the page without reloading the page.

For further reading on AJAX and using jQuery to perform AJAX, see the following tutorials, as well as this week's lecture will provide useful:

  • AJAX (general)
    • W3 Schools AJAX Tutorial
    • Another AJAX Tutorial
  • Using jQuery for AJAX
    • W3 Schools jQuery AJAX Tutorial
    • Another jQuery + AJAX Tutorial

Specifically, your implementation should allow users to comment & either see the comment added to the comments section without reloading the page, or respond with some error message of commenting failed. Rendering new HTML content will be easiest to do using jQuery, so actually rendering new content should not require very much code. You may find jQuery's built-in error handling hooks useful, particularly for commenting error cases such as when a user tries to comment without an internet connection or an internal server error occurs. For example:

var jqxhr = $.ajax( "example.php" )
    .done(function() {
       // handle success case ...
    })
    .error(function() {
       // handle HTTP error case ...
    })

How To Succeed With This Assignment

First and foremost, get comfortable with developing in your browser. Chrome is likely the most widely used for web development, and an excellent console + debugger comes with Chrome already. Firebug (an addon for Firefox) is also very popular, but likely more confusing if you are new to web development:

  • Google Chrome Developer Console
  • Firebug

Submission

Submit your final project proposal as a PDF via email to your moderator by 3:00pm, on Monday, October 29, 2012.

This assignment is due at the beginning of your discussion section the week of October 29, 2012. Please be sure to submit in SVN, and ask your moderator or TA before the deadline if you have any questions. Also, make sure to place your work in a folder called Assignment3.2, matching this spelling exactly.

For each week this assignment, your code should also be deployed to cpanel, and you should provide the URL to your moderator (for instance, in a README in your project). This is a required part of the assignment, and allows us to grade fairly. We will expect your project to be hosted at: 

http://web.engr.illinois.edu/~[netid]/

, which points to your public_html folder in cpanel.

Part IV: Load File Content Asynchronously

Next, use jQuery to fetch the contents of files via AJAX, so that the contents of source files are loaded without reloading the page.

Instead of requesting your SVN files from JavaScript, you should create another AJAX endpoint (PHP file) that will load the contents of a particular source code file from Subversion. If you still run into authentication issues trying to fetch the content of these files in PHP, feel free to create a copy of the SVN in your website, so that you can simply read the file from disk in PHP.

Cross-Domain Requests
It is important to note that directly making a request to your source code (i.e. to _https://subversion.ews.illinois.edu...) will not work. You will get an error like this in the JavaScript console of your browser:

XMLHttpRequest cannot load https://subversion.ews.illinois.edu/svn/fa12-cs242/... Origin http://... is not allowed by Access-Control-Allow-Origin.

Part V: Syntax Highlighting

Finally, use a syntax highlighting JavaScript library to provide syntax highlighting to the source code of your files. SyntaxHighlighter is probably the most widely used, but any similar library will suffice.

You may also want to allow users to invoke syntax highlighting on portions of their comments. This is not necessary for the assignment, but would be a simple extension of syntax highlighting for your source code files.

Dependency On Part IV
Although it may appear that you cannot complete this part without part IV, it is definitely possible. If you run into problems with Part IV, you can still accomplish this part by reading the contents of your source code files from the iframe directly, and dumping this into HTML using jQuery.

Testing

We realize that this week's assignment is difficult to test automatically, and so a manual test plan instead will suffice. A manual test plan should document the use of this week's features & the expected output. Screenshots of your site would be particularly useful for your manual test plan.
